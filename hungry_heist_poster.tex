\begin{posterbox}{Introduction}
Hungry Heist is a fast and easy-to-use platform for ordering food online. It connects customers with local restaurants and delivery riders using a modern and secure system.

\vspace{0.3cm}
The system is built to work for everyone. Each user—whether they are a customer, restaurant owner, or delivery rider—gets a custom dashboard that shows only the information they need to see.

\vspace{0.3cm}
It also includes safe online payments with Razorpay and live tracking for orders. This ensures a smooth and reliable experience for all users.
\end{posterbox}

\vfill

\begin{posterbox}{Problem Statement}
Many food delivery apps are confusing and don't work well for all users. The common problems include:
\begin{itemize}[leftmargin=1.2em, itemsep=0.1cm, label=\textbullet]
  \item Poor communication between restaurants and delivery riders.
  \item Unfair order cancellation rules that cost restaurants money.
  \item Hard-to-use screens for new restaurant owners.
\end{itemize}
\vspace{0.2cm}
Hungry Heist solves these problems with a clear design, fair fee rules, and a smart admin dashboard.
\end{posterbox}

\vfill

\begin{posterbox}{Objectives}
\begin{itemize}[leftmargin=1.2em, itemsep=0.15cm,label=\textbullet]
  \item Create a secure login system for all user types.
  \item Make it easy for customers to browse menus and order food.
  \item Add Razorpay so people can pay online safely.
  \item Give admins a dashboard to see reports and stats.
  \item Help delivery agents get orders and track their earnings.
\end{itemize}
\end{posterbox}

\begin{posterbox}{System Architecture}
Hungry Heist follows a three-tier architecture (Client, Server, and Database) for scalability and security.
\begin{itemize}[leftmargin=1.2em, itemsep=0.1cm, label=\textbullet]
  \item \textbf{Client Layer:} Built with React.js (Vite) using Axios for API communication and React Router for navigation.
  \item \textbf{Server Layer:} Uses Node.js and Express.js with a RESTful API design to handle logic and roles.
  \item \textbf{Middleware:} Includes JWT for security, bcrypt for passwords, and Multer for file uploads.
  \item \textbf{Database Layer:} Stores and organizes all system data safely using MongoDB and Mongoose.
\end{itemize}
\end{posterbox}

\vfill

\begin{posterbox}{Core Features}
\begin{itemize}[leftmargin=1.2em, itemsep=0.15cm,label=\textbullet]
  \item \textbf{Custom Dashboards:} Different screens for Customers, Restaurants, Agents, and Admins.
  \item \textbf{Smart Cart:} Automatically calculates the final price, including taxes and fees.
  \item \textbf{Order Tracking:} See exactly where the order is (Pending $\rightarrow$ Accepted $\rightarrow$ Picked Up $\rightarrow$ Completed).
  \item \textbf{Fair Cancellation:} Customers must pay a fee if they cancel an order after 60 seconds.
\end{itemize}
\end{posterbox}

\vfill

\begin{posterbox}{System Modules}
\begin{itemize}[leftmargin=1.2em, itemsep=0.15cm,label=\textbullet]
  \item \textbf{User Module:} Handles login, signup, and setting user roles.
  \item \textbf{Menu Module:} Lets restaurants add food and mark items out of stock.
  \item \textbf{Order Module:} Checks the cart and handles Razorpay payments.
  \item \textbf{Reports Module:} Creates charts and stats for the admin to track growth.
  \item \textbf{Feedback Module:} Allows customers to rate restaurants and riders.
\end{itemize}
\end{posterbox}

\begin{posterbox}{System Workflow}
\begin{itemize}[leftmargin=1.2em, itemsep=0.15cm ,label=\textbullet]
  \item \textbf{Login:} User logs in and access is granted based on their role.
  \item \textbf{Ordering:} Customers add food to their cart and go to checkout.
  \item \textbf{Payment:} System connects to Razorpay for secure online payment.
  \item \textbf{Processing:} Restaurant accepts order and prepares food.
  \item \textbf{Delivery:} Delivery agent picks up and completes the order.
  \item \textbf{Analytics:} Admin views earnings and order reports on the dashboard.
\end{itemize}
\end{posterbox}
